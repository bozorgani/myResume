# راهنمای بهینه‌سازی تصاویر

## مشکل کیفیت تصویر

اگر تصویر شما بی‌کیفیت به نظر می‌رسد، این راهنما به شما کمک می‌کند.

## راهکارهای بهبود کیفیت

### 1. استفاده از تصویر اصلی با کیفیت بالا

مهم‌ترین نکته این است که از یک تصویر اصلی با کیفیت بالا استفاده کنید:
- حداقل 1200x1200 پیکسل برای پرتره
- فرمت RAW یا JPG با کیفیت بالا (حداقل 90%)
- تصویر باید واضح و شارپ باشد

### 2. بهینه‌سازی تصویر با اسکریپت

از اسکریپت بهینه‌سازی استفاده کنید:

```bash
npm run img:opt /path/to/your/high-quality-image.jpg
```

یا:

```bash
node scripts/optimize-images.mjs /path/to/your/high-quality-image.jpg
```

این اسکریپت:
- تصویر را به فرمت WebP تبدیل می‌کند (کیفیت 90%)
- سایز مناسب برای هر استفاده را ایجاد می‌کند
- تصویر را بهینه می‌کند

### 3. بررسی فایل‌های موجود

اطمینان حاصل کنید که فایل‌های زیر در `public/images/` وجود دارند:
- `amin-bozorgani-portrait.webp` (1200x1200)
- `amin-bozorgani-portrait-512.webp` (1024x1024)
- `amin-bozorgani-portrait.jpg` (1200x1200)

### 4. نکات مهم

1. **همیشه از تصویر اصلی با کیفیت بالا شروع کنید**
   - اگر تصویر اصلی شما کوچک یا بی‌کیفیت است، نمی‌توانید کیفیت را بهبود دهید
   - حداقل 2000x2000 پیکسل برای بهترین نتیجه

2. **استفاده از Next.js Image Component**
   - کامپوننت Image به صورت خودکار تصاویر را بهینه می‌کند
   - از `quality={95}` برای بهترین کیفیت استفاده می‌شود
   - `priority` برای تصاویر مهم (LCP) استفاده می‌شود

3. **سایز مناسب**
   - width و height باید با سایز واقعی تصویر مطابقت داشته باشد
   - برای تصاویر دایره‌ای، از سایز مربع استفاده کنید

### 5. تست کیفیت

پس از بهینه‌سازی:
1. مرورگر را باز کنید
2. به صفحه اصلی و صفحه رزومه بروید
3. تصویر را در حالت zoom بررسی کنید
4. در DevTools، Network tab را باز کنید و تصویر را بررسی کنید

### 6. اگر هنوز کیفیت پایین است

1. **بررسی تصویر اصلی**: آیا تصویر اصلی شما کیفیت کافی دارد؟
2. **بازسازی تصاویر**: اسکریپت را دوباره اجرا کنید با تصویر اصلی
3. **افزایش کیفیت در اسکریپت**: در `optimize-images.mjs` می‌توانید `quality` را به 95 افزایش دهید
4. **استفاده از AVIF**: Next.js به صورت خودکار AVIF را برای مرورگرهای پشتیبانی‌کننده استفاده می‌کند

## نکات فنی

- WebP کیفیت بهتری نسبت به JPG در همان سایز فایل دارد
- Next.js به صورت خودکار تصاویر را برای دستگاه‌های مختلف بهینه می‌کند
- استفاده از `priority` برای تصاویر LCP (Largest Contentful Paint) مهم است
